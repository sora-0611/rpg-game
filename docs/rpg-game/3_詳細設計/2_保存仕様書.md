# 2_保存仕様書

> 変換元: `2_保存仕様書.xlsx`


## 保存仕様

| 列1 | 列2 | 列3 | 列4 |
| --- | --- | --- | --- |
| 保存対象 | 保存タイミング | 自動保存 / 手動保存 | セーブ失敗時の挙動 |
| セーブメタ情報 | すべての保存処理実行時 | 自動・手動共通 | 旧セーブを保持し、新規保存は破棄 |
| 現在の画面状態 | 重要イベント確定時 | 自動 | 状態は更新せず直前状態を維持 |
| 現在マップID | マップ入場完了時 / 退出確定時 | 自動 | マップ変更前の値を維持 |
| プレイヤー座標 | マップ入場完了時 / 退出確定時 | 自動 | 座標は前回保存値を使用 |
| 所持コイン | 戦闘結果確定時 / 強化成功時 | 自動 | コイン変動を反映せず元の値 |
| キャラHP | 戦闘結果確定時 | 自動 | HPは戦闘前の保存値に戻る |
| 装備強化段階 | 強化成功直後 | 自動 | 強化処理を無効として扱う |
| 所持アイテム | アイテム取得成功直後 | 自動 | アイテム未取得扱いに戻す |
| マップ取得済み状態 | アイテム取得成功直後 | 自動 | マップ状態を更新しない |
| イベント進行フラグ | イベント完了時 | 自動 | フラグは未達成のまま |
| 手動セーブ操作 | セーブで実行時 | 手動 | 保存前の値 |


## 保存目的

| 列1 | 列2 |
| --- | --- |
| 目的 |  |
| 本システムでは、探索・戦闘・強化などの重要イベント確定時のみ自動保存を行い、 |  |
| 中途状態を保存しないことでセーブ・ロード時のズレを防止する。 |  |
| 目的詳細 |  |
| セーブ後にロードした際、HP・コイン・強化段階・所持アイテムが一致すること |  |
| 画面遷移と内部データの不整合（表示は変わったが中身が戻る等）を防ぐ |  |
| 戦闘・強化・探索など複数機能を跨いだ状態復元を保証する |  |
| 仕様変更時でも既存セーブが破損しないようにする（バージョン互換） |  |
| 保存対象 | ズレ防止の観点での理由 |
| シーン | 画面表示と内部状態の不一致防止 |
| マップID | 別マップに飛ばされる事故防止 |
| 座標 | 探索再開位置のズレ防止 |
| コイン | 強化・報酬の二重反映防止 |
| HP | 戦闘結果の巻き戻り防止 |
| 強化段階 | ★表示と内部数値の不一致防止 |
| 所持アイテム | 再取得バグ防止 |
| マップ状態 | 取得済みアイテム再出現防止 |
| フラグ | ボス再出現・イベント再発防止 |
| エラー原因 |  |
| localStorage容量不足 |  |
| ブラウザ制限 |  |
| 保存処理中の例外発生 |  |
| 強制終了 |  |
| バージョン互換性（保存対象との関係） |  |
| 保存対象 | 互換対応 |
| 追加項目 | デフォルト値を補完 |
| 削除項目 | 読み飛ばす |
| 計算値 | 再計算する（保存しない） |
| 不明項目 | 無視してロード継続 |
