# Web RPG Game — Browser-based Prototype

ブラウザ上で動作するシンプルなRPGプロトタイプです。現在は `src/rpg-game` 以下でゲーム画面と主要ロジックを実装しています。

## 現在の実装状況

### 実装済み
- `src/rpg-game/index.html` に 8 画面分の画面構成が存在
- `src/rpg-game/css/` に画面表示と UI のスタイル
- `src/rpg-game/js/data.js` でキャラクター・アイテム・敵・マップ・強化コストなどのマスタデータ
- `src/rpg-game/js/state.js` でゲーム状態の初期化、状態補正、インベントリ、HP回復・ダメージ、強化判定・実行
- `src/rpg-game/js/save.js` で `localStorage` への保存/読み込み、破損チェック
- `src/rpg-game/js/item.js` で探索/戦闘におけるアイテム効果適用
- `src/rpg-game/js/battle.js` で戦闘ロジック、攻撃/防御/アイテム使用/逃走/勝利・敗北処理
- `src/rpg-game/js/explore.js` で探索移動、宝箱取得、敵遭遇、ボス戦開始、状態更新
- `src/rpg-game/js/ui.js` で画面レンダリング、ボタンイベント、画面遷移、UI 更新
- `src/rpg-game/js/main.js` でゲーム初期化とシーン管理

### 部分実装・注意点
- 探索画面のマップ描画はベースの座標管理があるが、視覚的な地図表示は限定的
- `つづきから` でのロード後は保存データを復元する処理はあるものの、復帰先は `MAP_SELECT` へ遷移する仕様になっている
- `localStorage` の保存は主に戦闘終了や逃走時に実行されるため、探索中のすべてのアクションで即座に自動保存されるわけではない

## 機能一覧

- ローディング画面表示
- タイトル画面: はじめから / つづきから / 設定
- マップ選択画面: 難易度1〜3マップの選択
- 探索画面: 上下左右で移動、宝箱取得、ランダム/固定エンカウント
- 戦闘画面: 攻撃、防御、アイテム、逃走
- 強化画面: 所持コインによるキャラクター強化
- アイテム画面: 所持アイテム一覧表示、探索時使用
- セーブ/ロード: `localStorage` に保存・読み込み
- 設定画面: メニュー遷移用のプレースホルダー

## 手動テスト項目

### 画面遷移
1. ローディング画面が表示され、数百ミリ秒後にタイトル画面に自動遷移する
2. タイトル画面の「はじめから」「つづきから」「設定」ボタンが存在する

### 新規プレイフロー
1. 「はじめから」を押してマップ選択画面に移動する
2. マップ1を選択して探索画面へ移動する
3. 探索画面で上下左右キーを押して移動を試す
4. 宝箱を取得した場合、メッセージ表示とインベントリ増加を確認する
5. 敵に遭遇した場合、戦闘画面へ遷移する

### 戦闘
1. 戦闘画面で「戦う」「防御」「アイテム」「逃げる」ボタンを押す
2. 攻撃で敵HPが減り、敵ターンが発生することを確認する
3. 防御を押すと次の敵攻撃が軽減される
4. 戦闘アイテムを開いて使用し、効果が適用されることを確認する
5. 通常敵戦では逃走が成功/失敗する挙動を確認する
6. ボス戦では逃走不可であることを確認する

### アイテム/強化
1. 宝箱などで入手したアイテムが所持に追加される
2. 探索中に使用可能なアイテムを使用してHP回復が行える
3. 使えないアイテムは戦闘専用か探索専用として扱われる
4. 強化画面でキャラクターを選択し、所持コインが足りない場合は強化できない
5. 強化成功後に所持コインが減り、能力値が更新されることを確認する

### セーブ/ロード
1. 戦闘終了後または逃走後に `localStorage` にセーブされる
2. 画面をリロードして「つづきから」を選択すると保存データが読み込まれる
